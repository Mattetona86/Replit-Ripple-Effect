--------------------------------------------------
11. CONTROLLO DELLE FORMULE
--------------------------------------------------

Verificare e rendere esplicite le formule.

Operating Margin =
Operating Income TTM / Revenue TTM

Net Margin =
Net Income TTM / Revenue TTM

FCF =
Operating Cash Flow TTM - Capital Expenditure TTM

FCF Margin =
FCF TTM / Revenue TTM

ROIC =
NOPAT / Average Invested Capital

NOPAT =
Operating Income TTM × (1 - normalized tax rate)

Invested Capital =
Equity + Interest-Bearing Debt - Cash

ROE =
Net Income TTM / Average Equity

ROA =
Net Income TTM / Average Total Assets

P/E =
Current Adjusted Price / Diluted EPS TTM

Price/FCF =
Market Cap / FCF TTM

EV/EBITDA =
Enterprise Value / EBITDA TTM

Per ogni ratio:

- non mescolare numeratore TTM con denominatore annuale;
- usare average balance-sheet values per ROE, ROA e ROIC;
- distinguere reported values da calculated values;
- mostrare N/M quando il denominatore è negativo o vicino a zero;
- non mostrare percentuali di crescita ingannevoli quando il valore iniziale è negativo;
- non trattare dati mancanti come zero.

Aggiungere test automatici per tutte le formule.

15. ANALISI AI
--------------------------------------------------

L’AI non deve ricevere raw Yahoo data.

Deve ricevere un JSON già validato e calcolato:

{
  "company": {},
  "businessTrend": {},
  "priceVsBusiness": {},
  "valuation": {},
  "financialStrength": {},
  "newsMomentum": {},
  "catalysts": [],
  "risks": [],
  "dataConfidence": {},
  "sources": []
}

L’AI deve:

- interpretare;
- sintetizzare;
- confrontare;
- indicare incertezza.

Non deve:

- calcolare ratio;
- inventare competitor;
- inventare cause;
- inventare rischi qualitativi;
- fornire target price;
- fornire raccomandazioni buy/sell.

Output massimo:

- headline: 12 parole;
- summary: 80 parole;
- 3 catalysts;
- 3 risks;
- 3 metrics to monitor.

17. GESTIONE DEI LIMITI DI YAHOO FINANCE
--------------------------------------------------

Non tentare di riempire i dati mancanti inventandoli.

Creare una matrice di disponibilità:

- current fundamentals;
- historical fundamentals;
- current valuation;
- historical valuation;
- prices;
- news;
- analyst estimates;
- peer data.

Se Yahoo Finance non fornisce dati sufficienti per:

- valuation history;
- peer medians;
- news momentum;
- estimates;

creare un provider interface che permetta di aggiungere successivamente un provider diverso.

Esempio:

interface MarketDataProvider {
  getCompanyProfile(symbol)
  getCurrentPrice(symbol)
  getHistoricalPrices(symbol)
  getFinancialStatements(symbol)
  getKeyMetrics(symbol)
  getHistoricalMetrics(symbol)
  getPeers(symbol)
  getNews(symbol)
  getAnalystEstimates(symbol)
}

L’interfaccia deve continuare a funzionare anche quando alcuni provider non sono configurati.
